# Project map (Cartographer)

Purpose
- Provide a single canonical map of repository structure (modules, packages, services, important files) generated by Cartographer so the assistant and contributors can quickly understand the project.

Generate
- Run your Cartographer command and save output to docs/project-map.json or docs/project-map.yml. Example (adjust to your Cartographer setup):
  cartographer generate > docs/project-map.json
- Prefer JSON or YAML for machine-readability.

Placement & naming
- Place the generated file at: docs/project-map.json or docs/project-map.yml
- Cartographer also commonly emits an HTML visualization; place it at docs/project-map.html if present.
- The assistant looks for docs/project-map.{json,yml,yaml,html} when analyzing the repo.

Using Cartographer HTML (mermaid)
- If docs/project-map.html exists, the assistant will extract the mermaid graph block (a <div class="mermaid">...</div>) and parse node and edge lines.
- Expected node label format in mermaid: n1["NAME\\nPATH"] where NAME is a human label and PATH is the filesystem path.
- Expected edge format: n1 --> n2 (the assistant will infer relationships: contains when paths imply containment, otherwise depends_on).
- Keep the mermaid graph focused: high-level modules and important files are sufficient for tool-driven analysis.

Minimal expectations
- Nodes: name, path, type (module/library/service)
- Edges: from, to, relationship (depends_on, contains)
- Optional metadata: language, entrypoint, tests, owner

Example (JSON snippet)
```
{
  "nodes": [
    {"name":"api","path":"src/api","type":"service"}
  ],
  "edges": [
    {"from":"api","to":"db","relationship":"depends_on"}
  ]
}
```

How the assistant will consume it
- On project work, the assistant will load docs/project-map.* to: identify modules, find likely change locations, suggest targeted edits, and map high-level tasks to files.
- Keep the map up-to-date after large refactors to ensure recommendations remain accurate.

Optional Copilot config snippet (.copilot/project-map-config.yml)
```
project_map:
  path: docs/project-map.json
  format: json
  refresh_command: "cartographer generate > docs/project-map.json"
  auto_refresh: false
```

Best practices
- Regenerate after major restructuring or dependency changes.
- Keep the map focused: include high-level nodes and important files — avoid listing every generated/third-party file.

If helpful, the assistant can add a CI job or npm script to regenerate the map automatically.
